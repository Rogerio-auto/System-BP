# Handoff IA para atendimento humano

## O que é handoff

Handoff é o processo pelo qual a Ana Clara transfere uma conversa para um atendente humano. Isso acontece automaticamente em duas situações:

1. **Decisão da Ana Clara** — ela identificou que o caso precisa de atendimento humano (por exemplo: pergunta fora do escopo, situação complexa ou solicitação explícita do cidadão).
2. **Falha técnica** — o agente de IA não conseguiu processar a mensagem por indisponibilidade momentânea.

## O que o cidadão recebe

Quando o handoff é acionado, o cidadão recebe automaticamente a mensagem:

> "Olá! Um atendente vai te responder em instantes. Aguarde um momento. 🙂"

A mensagem é enviada pelo sistema e não expõe detalhes técnicos. Mesmo que a fila processe a mensagem mais de uma vez, o cidadão recebe o aviso apenas uma vez.

## O que acontece no inbox

Após o handoff:

- A conversa muda para o status **Pendente** (aguardando atendente humano).
- O inbox do workspace é atualizado automaticamente em tempo real.
- Um atendente humano pode assumir a conversa clicando nela no inbox.

## Como o operador identifica uma conversa em handoff

No inbox, conversas em handoff aparecem com status **Pendente**. Localize-as usando o filtro **Pendentes** na parte superior da lista de conversas.

<Callout type="tip">
  Ao abrir uma conversa em handoff, você pode ler todo o histórico da triagem feita pela Ana Clara no painel de mensagens antes de responder ao cidadão.
</Callout>

## Limitações atuais

- **Atribuição automática de agente:** o handoff marca a conversa como pendente, mas não a atribui automaticamente a um atendente específico. A distribuição entre atendentes é planejada para uma versão futura.
- **Notificação dedicada:** a conversa aparece no inbox como pendente. Uma notificação push dedicada para o atendente está planejada para versão futura.

## Referência técnica (para administradores e devs)

O fluxo técnico resumido:

```
Mensagem do cidadão
  → Worker de IA (fila interna)
    → Agente de IA: falha ou handoff sinalizado
      → Envia mensagem de aviso ao cidadão (idempotente)
      → Atualiza status da conversa para 'pending'
      → Notifica o inbox em tempo real
      → Registra auditoria (ação 'livechat.ai_handoff')
```

Arquivos de implementação:
- `apps/api/src/modules/livechat/ai-handoff.ts` — módulo de handoff.
- `apps/api/src/workers/livechat-ai.ts` — worker que aciona o handoff.
