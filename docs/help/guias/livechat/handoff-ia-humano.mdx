# Handoff IA → Humano no LiveChat

## O que é handoff

Handoff é o processo pelo qual o agente IA do livechat transfere uma conversa para um
atendente humano. Isso acontece automaticamente em duas situações:

1. **Falha técnica da IA** — o LangGraph não conseguiu processar a mensagem (timeout, erro interno).
2. **Decisão do grafo** — o agente IA identificou que o caso precisa de atendimento humano
   (ex.: pergunta fora do escopo, situação complexa, solicitação explícita do cidadão).

## O que o cidadão recebe

Quando o handoff é acionado, o cidadão recebe automaticamente a mensagem:

> "Olá! Um atendente vai te responder em instantes. Aguarde um momento. 🙂"

A mensagem é enviada pelo sistema (bot) e não expõe detalhes técnicos do erro.
Ela é **idempotente** — mesmo que a fila reprocesse o job, o cidadão recebe a mensagem
apenas uma vez.

## O que acontece no inbox

Após o handoff:

- A conversa muda para o status **Pendente** (aguardando atendente humano).
- O inbox do workspace é atualizado em tempo real via WebSocket.
- Um agente humano pode assumir a conversa clicando nela no inbox.

## Como o operador identifica uma conversa em handoff

No inbox, conversas em handoff aparecem com status **Pendente**.
O motivo do handoff (`ai_unavailable` ou `ai_requested`) fica registrado no audit log
e pode ser consultado por administradores.

## Fluxo técnico resumido (para devs)

```
Mensagem do cidadão
  → Worker livechat-ai (hm.q.livechat.ai)
    → LangGraph: falha ou handoff.required=true
      → triggerLivechatHandoff(db, { reason, conversationId, messageId })
        1. sendMessage(fallback) — idempotency key: ai_fallback_<messageId>
        2. UPDATE conversations SET status='pending'
        3. publish(socketRelay, conversation:updated)
        4. auditLog(action='livechat.ai_handoff')
```

## Arquivo de implementação

`apps/api/src/modules/livechat/ai-handoff.ts` — módulo de handoff.
`apps/api/src/workers/livechat-ai.ts` — worker que chama `triggerLivechatHandoff`.

## Limitações atuais

- **Atribuição de agente**: o handoff marca a conversa como `pending` mas não atribui
  automaticamente a um agente específico. A distribuição (round-robin, skills) é planejada
  para um slot futuro.
- **Notificação de agentes**: a conversa aparece no inbox como pendente. Notificação push
  dedicada é planejada para slot futuro.
